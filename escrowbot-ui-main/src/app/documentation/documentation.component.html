Dashboard Page

  1. On completing the steps in Authentication above, The user is presented the Dashboard page, Note that all Information displayed on a page for a user pertains only to his User Hierarchy e.g Regional ,Business Hub, Service Center User

  2. For example a User With Regional Designation for Oyo Will see all Customer data Within Oyo State, A business Hub user designated for APATA will only see records for Apata business hub

  3. Some cards on the keyboards are clickable to reveal more detailed data as to what they represent, some cards like the Online Feeders and Operation statistcs are not clickable, Prepaid , Postpaid and Today's Payments Cards are all clickable

  5. Postpaid card shows the total of all available customers pertaining to the users locaton hierarchy either Region, Business Hub or Service Center

  11. The collection statistics graph can be zoomed in/out to view detailed dates and scrolled to view more timeframe

Searching for customers is far more efficient and less time consuming than tranversing the table looking for record

2: Deep Searching: This involves typing the value to be searched and searching by a particular attribute e.g Name, Account No, Region (For HQ Users), Business Hub , Service Center etc

3: Quick Searching using the method described previously

Customer Creation/Update Process

Customer Creation Process


To be able to create a new Customer you must have Can Create Customer permission or your user role must be designated for customer creation



1: Click the Create Customer button in the Customers List page as described here

2: In the side-drawn modal fill in the customer information for each tab present (Basic Information, Account Information, Location Information etc).

If all madatory data (as per NERC requirements) to Immediately create the customer are not available at that instant, use the save button for each tab to save your progress for each customer information. As you save a draft is created for each customer will can be loaded when you have more data to fll in. It helps you pick off from where you stopped for each customer you intend to create


Below is a little prep on drafts before proceeding to actual customer creation


Incorrect Email and Phone Number in Basic Information Form (Saving to Draft)

When saving to draft If invalid email or phone number is entered, you will get an error prompting you to enter valid data (Next Slide)

Valid Email and Phone Number

When valid email and phone number are entered , if successful new draft will be created

Draft Created, clicking on the arrow highlighted row will load all the data we saved so far into the form (completely overwriting any data previously in the form) and we can continue adding information to the draft from where we left off

From this point on , a valid customer will be created with all mandatory data using drafts to save each form as we proceed

Previous Next

Note: Even though a draft can be created from any of these tabs by clicking the save button on any tab, it is highly recommended to create a draft from the basic information tab ,entering atleast email and phone number


To Create a new Customer the following steps should be followed


1: Fill in the Basic Information Form to whatever extent and click the orange Save to draft button (Vaild email and Phone Number are required in this tab)



2: Fill in the Account Information Form to whatever extent and click the orange Save to draft button



3: Fill in the Location Information Form to whatever extent and click the orange Save to draft button



4: Fill in the Assets Information Form to whatever extent and click the orange Save to draft button, the DSS and Feeder field are related, click the DSS Owner Dropdown and select or search for a DSS owner with the search input in the dropdown


The DSS Name dropdown is populated based on what is selected as the DSS Owner,trying to select a DSS Name before selecting a DSS Owner will Show you an error.

Once the DSS Name is selected, the feeder type and Feeder name fields are auto filled if the a corresponding Feeder for the DSS Name was found, If no DSS Name Matches a Feeder the Feeder Type and Feeder Name will remain blank,click the save button to Save to Draft



5: Fill in the Property Type Information Form to whatever extent and click the orange Save to draft button



6: At this point if all NERC & KYC fields are filled in and contain valid data in all form, click the Submit button to Post this data for further review and Approvals by relevant parties. The first relevant party to confirm will receive a mail
If there are any invalid data in the form or required fields missing data you will be notified once you try submitting.


If you do not wish to submit right away, you can always load the data (as earlier explained) from draft again to submit for Approvals.


On submitting and confirming Submission for Approval, if a an Approval awaiting customer record previously if found to have either the same Phone Number, Email Address or Account Number with your submission record a modal like shown below will pop up



The Email Address Table at the top has no Item , this means no existing record in Approval Queue was found with the email that was just used while filling the form earlier



The Mobile Number table contains two record in the approval queue sharing the same phone number, you can chose to update any of those two records by simply clicking on the record you wish to Update. This feature is useful if you want to update the information for a customer you already submitted earlier. Note that you can only update a record in this manner only if the Approval process has not started. Clicking the row and confirming the update will update the existing Awaiting Record with data you just submitted


If however you choose to create a new record sharing same either email or phone number click the Create New Record at the bottom of the popup. This will queue a fresh Awaiting Customer with the same Email or Phone as the records in the email or phone number tables on the modal


The Account Number Table on this Modal Popup is only relevant when Editing an Already Created Customer having an Account number already

Customer Record Approval Process (CRMD)

To approve an Awaiting Customer or Edited customer your account must have either the Can Approve Customer Creation permission or the role assigned to your account should


To open the Customer Approval page, expand the Customers menu in the Navigation bar and Click on Customer Approvals. This Menu item is only visible if you have the correct permissions to Approve Awaiting Customers.


For the purpose of this documentation BHM will be assigned as the First Awaiting customer Approver, the Next in line will be the ADT


It is note worthy that only after the BHM has approved the Awaiting Customer will the ADT be able to see such a record. If the BHM declines the record, the ADT will not see this record at all. The account that created this record can resubmit after performing the neccessary correction


For the previously submitted Awaiting customer in the previous section (Red Arrow), the record is seen by the BHM for that region as in the image below


On this page, you will notice a Status Filters Dropdown. Hover on the dropdown to reveal 3 status types: Pending, Approved and Declined. Clicking on any of them will filter the table based on the filter selected. If Approved is clicked only previously approved awaiting customers will show up in the table.


The search bar at the top of the page functions in exactly same fashion as described in the Customer list page


On hovering on any record in this table , a view details icon will show up. Click on it to view all Submitted information about the record. Clicking on this will reveal a vertically scrollable side-drawn modal read-only form as shown below with the records data grouped based relation.



Scroll this form to view all data submitted for the record, at the extreme bottom of the form is a Comments and Observation Section (Black Arrow). Here you can state a reason for declining this Awaiting Customer or write a comment .




To Approve a the record click the Green Approve Button and to Decline with a reason/comment click the Yellow Decline Button. Close the form with the Exit button


If the record is successfully approved the next in line in our case the ADT will receive a mail requesting for approval. Only at this point will the ADT be able to see this record in his Customer Approval page. If the record is declined, the Account that created the Awaiting Customer will get a mail notification.



The next approver in our case here the ADT will repeat this exact process starting from here. After succesfull approval, the awaiting customer is created in the customers database as a new customer. If this process was for an already existing customer ,the customer record will be updated and the BHM will get a mail notification



Customer Account Adjustment Document (CAAD)

Note: CAAD request creation is only possible if a customer has a Region, Business Unit and a Service Center. If any is unavailable an error modal will popmup to indicate this error.


To create a new CAAD request for a customer click on the customer to navigate to the Customer Details page as earlier discussed here. Click the New CAAD tab (Blue arrow) , the page below will be seen



The highlighted sections below will be used to explain how they integrate to create a new caad request.



Some basic information about the customer is shown in the basic information section.


Red Boundary: This section contains widgets to select the VAT (Left- Dropdown Field) and The Estimated Amount (Right- A Number Input Field) to use for this Caad Request. They are both required fields


Sky Blue Boundary: This Add New LineItem button is only visible to those who have Can Create CAAD permission or to a role that has been assigned such a task. This contains a button to add CAAD request Line Items, There is currently no limit to the number of line Items that can be created. Clicking on this button will add a new row to the table each time.


Green Boundary: This Submit button is only visible to those who have Can Create CAAD permission or to a role that has been assigned such a task. After all line items are added and the computation is complete this button is used to submit the CAAD Request for approvals


Blue Arrow: This text field is only visible to those who have Can Approve CAAD permission or to a role that has been assigned such a task. This text field is for comments in the event that whoever is approving declines the caad request. if declining they have to Mandatorilytype a comment in the text box


Pink Boundary: This boundary contains the Approve and Revert buttons and are only visible to those who have Can Approve CAAD permission or to a role that has been assigned such a task


Purple Boundary: This area shows the resultant refund amount, after all computations have be done
