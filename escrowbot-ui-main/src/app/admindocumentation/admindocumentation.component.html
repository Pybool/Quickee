    Create & Update Account Types


    To create a new Account Type option (Prepaid & Postpaid), click the Add button for the Account Type section and a modal shown below pops up


    Type in the new Account Type you desire to add in the input box and Press Enter on the keyboard to Enlist it , you can add multiple at one go pressing enter for each to enlist. After enlisting all click the save Button .In the Image below we type in Testprepaid in the Input Box and press enter to enlist it. When enlisted it appears as a lime green pill on the modal, repeat for each account type to add several and click save button. You can delist by clicking the X button on the pill component.
    Upon successful creation of the Account types you receive a notification


    To Update an existing Account Type, click the Edit button for the Account Type section and a modal shown below pops up



    Double click on the pill you wish to edit and type in the text that appears in the text input that appears, Press the enter button, the Input box will revert to a Pill updated with new name for the Account Type Option and click save.You will get a notification if successful


    For cases where an account type you have created is already in use ,Instead of deleting account Types, edit them as the records using them will automatically inherit the updated account type. Deleting will cause orphaned account type option customer records. To delete an Account Type in Editing click the X icon on a pill. (This is Discouraged is records are already bearing the account type, edit instead.)



    After editing ensure to click save button

    Manage Users

    This section deals with managing users accounts

    Expand the admin section on the navigation bar and click on Manage Users link, the page below will be displayed.



    Create New User



    There are two types of Admin users 1: Base Admin Users 2: Elevated Admin User (only one user bears this account and will be created upon request from Ibedc Team)


    For an admin user without Base Admin User , the modal appears with some missing 2fa widgets, the Elevated Admin User is able to propagate or assign other users with the ability to manage 2FA operations


    The Image below is what is seen by Base Admin Users



    The Image below is what is seen by Elevated Admin Users



    Click the Create button as seen on the page above to create a new user. A side-drawn modal will pop out as seen below



    Fullname and Email fields are mandatory fields, Enter a default password e.g 0000 or any password at all and confirm the password.
    In the Positions dropdown select the position this user will operate as.
    Next for the can create dropdown Select an option.
    If Cant create users is selected the new user will not be able to create other users.
    If Can create is selected , this user account will have permission to create other users.
    For the permissions checkboxes
    If Can create/Update is selected , this user account will have permission to create and edit other Customers.
    If Can Approve Customer Creation is checked , this user account will have permission to Approve Customer Creation and Updates (CRMD) .
    If Can Approve CAAD is checked , this user account will have permission to Approve CAAD Requests (CRMD) .
    If Can Approve Manage 2FA is checked , this user account will have permission to manage 2fa on other users account .


    In the radio buttons below select the level at which the user will operate
    If Head Quarters is selected , this user will be marked as an HQ user having the ability to view records on the cms from all regions and lower hierarchies.
    If Regional is selected , a dropdown appears select the region under which this user will operate, this user will have the ability to view records on the cms from only that chosen region and lower hierarchies.
    If Business Unit is selected , 2 dropdowns appears select the region first then business hub under which this user will operate, this user will have the ability to view records on the cms from only that chosen Business Unit and lower hierarchies.
    If Service Center is selected , 3 dropdowns appears select the region first then business hub followed by the service center under which this user will operate, this user will be marked as an Service Center user having the ability to view records on the cms from that particular service center


    Select appropriate permissions based on admin type and click create user


    Edit Users



    To disable/Enable a user toggle the user Icon (Red highlited)


    To disable/Enable multiple users Select the users with the checkboxes at the far left of each record and click the Actions Dropdown (Blue highlited) and select an action to perform. Either Enable or Disable


    To edit a user hover on a record as shown below in the image, click the edit Icon (Black highlighted) and a similar modal as the Create User Side-drawn modal will pop out with all available user's details pre-filled in the form. Edit the form as neccesary and click Update User button to save.



    2FA Authentication


    To Enable 2FA on a user's account check the Enable 2FA Checkbox at the bottom of the form, Uncheck to disable 2FA Authentication for the user.


    To Reset a User's Account with 2FA Steps already completed, click the red Clear 2FA Credentials button at the bottom of the page. The user will have to rescan the QRCode as described in Authentication section of the Users Guide.
